# 环境搭建

## 安装 JDK

1. **下载 JDK**：访问 [Oracle 官网](https://www.oracle.com/java/technologies/javase-jdk11-downloads.html) 或 [OpenJDK](https://jdk.java.net/) 下载适合你操作系统的 JDK 安装包。

2. **安装 JDK**：运行下载好的安装程序，按照提示完成安装。注意选择安装目录。

3. **配置环境变量**：
   - 在 Windows:
     - 右键点击“此电脑” -> “属性” -> “高级系统设置” -> “环境变量”。
     - 在系统变量中，点击“新建”，变量名为 `JAVA_HOME`，变量值为 JDK 的安装路径（例如 `C:\Program Files\Java\jdk-11.0.10`）。
     - 找到`Path`变量，点击“编辑”，添加新路径 `%JAVA_HOME%\bin`。
   - 在 macOS/Linux:
     - 打开终端，输入 `echo 'export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk-11.0.10.jdk/Contents/Home' >> ~/.bash_profile`
     - 输入 `echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.bash_profile`
     - 执行 `source ~/.bash_profile`。

4. **验证安装**：在命令行输入 `java -version`，如果出现对应版本号说明安装成功。

## 安装 Maven

1. **下载 Maven**：访问 [Maven 官网](https://maven.apache.org/download.cgi) 下载最新版本的 Maven 压缩包。

2. **解压安装包**：将压缩包解压至目标目录，例如 `C:\apache-maven`。

3. **配置环境变量**：
   - `MAVEN_HOME`：和上面类似，设置为 Maven 的安装路径。
   - `Path`：添加 `%MAVEN_HOME%\bin`。

4. **验证安装**：在命令行输入 `mvn -v`，如果能看到 Maven 的版本信息，说明安装成功。

## 安装 Node.js

1. **下载 Node.js**：访问 [Node.js 官网](https://nodejs.org/) 选择合适的安装包下载。

2. **安装 Node.js**：运行安装包，按照提示完成安装。建议勾选“Add to PATH”。

3. **验证安装**：在命令行输入 `node -v` 和 `npm -v`，查看版本信息以确认安��成功。

## 安装 MySQL

1. **下载 MySQL**：访问 [MySQL 官网](https://dev.mysql.com/downloads/mysql/) 下载适合你操作系统的版本。

2. **安装 MySQL**：运行安装程序，按照向导完成安装，选择服务器类型和配置方式。

3. **安全配置**：设置 root 用户密码，移除匿名用户，禁止远程 root 登录等。

4. **验证安装**：使用 `mysql -u root -p` 登录 MySQL，如果能够成功 login，说明安装成功。

## 安装 Redis

1. **下载 Redis**：访问 [Redis 官网](https://redis.io/download) 下载适合你操作系统的 Redis。 

2. **安装 Redis**：
   - 在 Windows 上可以使用 [Memurai](https://www.memurai.com/) 或 [Redis for Windows](https://github.com/MicrosoftArchive/redis/releases)。
   - 在 macOS，可以使用 `brew install redis`。

3. **启动 Redis**：运行 `redis-server` 启动 Redis 服务。

4. **验证安装**：在另一个终端窗口中输入 `redis-cli ping`，应该返回 `PONG`。

## 常见错误及解决方案

- **JDK 找不到**：检查 `JAVA_HOME` 和 `Path` 是否设置正确。
- **Maven 无法启动**：检查 `MAVEN_HOME` 和 `Path` 设置是否正确。
- **Node.js 安装后未识别**：确保 Node.js 安装目录在 `Path` 中。
- **MySQL 登录失败**：确保使用正确的用户名和密码，且 MySQL 服务正在运行。
- **Redis 无法启动**：查看启动日志，检查是否有端口冲突或文件权限问题。
